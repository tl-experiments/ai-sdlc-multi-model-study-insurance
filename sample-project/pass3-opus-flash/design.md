## Data model (Prisma)
See `prisma/schema.prisma`. Models: `Employee` (with self-relation `reports_to`), `TimeEntry`, `LeaveRequest`, `LeaveBalance`, `AuditLog`. PII columns suffixed `_ct` hold AES-256-GCM ciphertext.

## Module structure
`src/common/*` — encryption util, guards, decorators, audit interceptor, error filter, request-id middleware. `src/auth/` — JWT login. `src/audit/` — audit log read. `src/employees/`, `src/time-entries/`, `src/leave-requests/`, `src/reports/` — feature modules.

## ADRs
- **ADR-001** AES-256-GCM envelope encryption (per-record DEK wrapped by env-supplied KEK).
- **ADR-002** Audit log immutability (no UPDATE/DELETE in code; auditor-only read).
- **ADR-003** Role-based masking via single utility `maskEmployee()`.

common → auth → audit → employees → time-entries → leave-requests → reports → tests → docs.
