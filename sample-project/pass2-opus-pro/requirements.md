# Requirements — Workforce Operations Service (Pass 1: Opus-only)

> Auto-generated by the AI-SDLC orchestrator from `../brief.md`. Approved at HITL Gate 1.

## In scope
1. Backend REST API + frontend web UI.
2. Five resource domains: Employees, Time Entries, Leave Requests, Reports, Auth + Audit.
3. Persistent storage via Prisma + SQLite (POC).
4. Local password auth + JWT; four roles (admin, manager, employee, auditor).
5. PII encryption at rest for `government_id`, `bank_account`, `salary_base`.
6. Role-based response masking on PII fields.
7. Append-only audit log of all PII reads/writes.
8. OpenAPI / Swagger docs at `/docs`.

## Out of scope
SSO/SAML/OAuth, background jobs, file uploads, multi-tenancy, prod deploy artifacts.

## PII inventory
| Field | Sensitivity | Protection |
|---|---|---|
| `full_name`, `email` | Low | Visible per role |
| `phone`, `address` | Medium | Masked from peer-role |
| `government_id`, `bank_account`, `salary_base` | High | Encrypted at rest; masked except admin/self |

## Role matrix
| Resource | admin | manager | employee | auditor |
|---|---|---|---|---|
| employees / read all | full | reports-only, masked | self only | masked |
| employees / write | yes | no | self (non-PII) | no |
| leave-requests / approve | yes | reports' only | no | no |
| audit_log / read | no | no | no | yes |
